## Implementation Summary

### DmanDatasetBuilder (pyclass, unsendable)
- `__new__(name, schema_path=None)` — opens in-memory DB, registers dataset
- `add_image(file_name, width=None, height=None)` — stages image
- `add_annotation(image_file_name, category_name, bbox, score=None)` — stages annotation
- `set_category(name, supercategory=None)` — stages category
- `build()` → DmanDataset — commits all staged data in a single transaction; rolls back on any error

### DmanDatasetUpdater (pyclass, unsendable)
- `new_with_db(dataset_name, db_path)` — loads existing dataset from DB path
- `add_image(file_name, width=None, height=None)` → -1 sentinel
- `add_annotation(image_file_name, category_name, bbox, score=None)` — stages annotation
- `remove_image(file_name)` — stages removal
- `apply()` → DmanDataset — applies all staged ops in a transaction; rolls back on error

### Python functions registered
- `create_dataset(name, images, annotations, categories=None, schema_path=None)` → DmanDataset
- `update_dataset(dataset_name, db_path, add_images=None, add_annotations=None, remove_images=None)` → DmanDataset

### Transaction pattern
Both `build_internal()` and `apply_internal()` use:
  BEGIN IMMEDIATE → operations → COMMIT / ROLLBACK on error
